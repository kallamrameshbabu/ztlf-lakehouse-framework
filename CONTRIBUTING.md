# Contributing

This repository supports an academic reference implementation.

Before contributing:

1. Open an issue describing the proposed change.
2. Do not submit credentials, proprietary code, confidential architecture,
   employer-owned material, or unlicensed data.
3. Keep validation behavior deterministic and covered by tests.
4. Document any change that affects experimental record counts.
5. Preserve a clear distinction between implemented controls and proposed
   future controls.

By contributing, you confirm that you have the right to submit the
material. A formal contributor license policy may be added after the
repository license is selected.

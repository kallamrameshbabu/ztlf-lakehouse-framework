# Public Release Checklist

- [ ] Remove tenant IDs, subscription IDs, workspace URLs, and storage paths.
- [ ] Remove secrets, tokens, account keys, connection strings, and SAS URLs.
- [ ] Confirm no Microsoft or employer-owned source code is included.
- [ ] Confirm all code was authored for this independent research project.
- [ ] Verify dataset provenance and redistribution rights.
- [ ] Replace the source dataset with synthetic or clearly licensed data.
- [ ] Record actual runtime and package versions.
- [ ] Add unit tests for all seven quality rules.
- [ ] Add expected record-count assertions.
- [ ] Verify notebooks run from a clean environment.
- [ ] Select and add an appropriate software license.
- [ ] Review figure ownership and licenses.
- [ ] Replace placeholder GitHub URL in `CITATION.cff`.
- [ ] Update article DOI and publication metadata after acceptance.
- [ ] Create a signed release tag, such as `v1.0.0`.
